# dpas-lab.pages.dev — 스타일 분석 노트

이 사이트가 잘하는 것을 정리한 메모. Rust 위키도 같은 원칙을 따른다.

## 인덱스 페이지

- 한 화면에 학습 단계 전부가 보인다 (스크롤 1~2번이면 끝).
- 각 단계가 **번호 있는 카드**로 표시. 카드 안에는 (1) 제목 (2) 한 줄 설명 (3) 작은 태그(2~4개).
- 카드 자체가 클릭 가능 영역. 호버 시 미세한 강조.
- 상단에 과목명/연도를 작게 두고, 큰 제목은 사이트 자체의 정체성("I/O Completion 실습").

## 단계 페이지

- 페이지 상단: "← 목차로 돌아가기" 단일 링크.
- 본문 폭이 좁다(약 700~800px). 긴 줄을 강제로 막아 가독성을 높임.
- 코드 블록은 옅은 회색 배경 + 1px 테두리. 화려하지 않다.
- "왜 커널 5.18인가?" 같은 질문 형식의 소제목을 자주 사용 — 학습자가 가질 의문을 선점.

## 환경 분기

- Step 6(실습 환경 설정)에서 macOS / Linux / Windows를 **세 개의 큰 카드**로 나눠 별도 페이지로 보낸다.
- "공통 요구사항"을 카드 아래에 따로 뽑아둔다.
- 경고는 강조 박스(VirtualBox 주의)로.

## 가져올 점 / 바꿀 점

가져올 점:
- 번호 카드 인덱스 구조
- 좁은 본문 폭
- 한 단계 = 한 페이지
- OS 분기를 1급 시민으로 취급

바꿀 점:
- dpas-lab은 OS별로 별도 페이지를 만들지만, Rust 위키는 **한 페이지 안의 탭**으로 처리한다 (모듈이 짧기 때문에 페이지를 더 쪼개면 산만해짐).
- 강조색은 Rust 오렌지(#ce422b)로 통일.
